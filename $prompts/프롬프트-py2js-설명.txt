지금부터 내가 너에게 아래와 같은 형태로 컨텐츠를 올릴거야.
프로그래머스의 코딩 테스트 문제의 내용과 내가 해결한 파이썬 소스코드야.
오늘 수업 내용은 구간합과 슬라이딩윈도우야.
내가 푼 문제를 학생들에게 교안으로 제작해서 제공하려고 해.
너는 학생들을 위한 선생님의 역할을 맡아줘.
-----------------
문제 URL

[문제를 해결하기 위해 내가 작성한 파이썬 소스코드]
----------

우선 파이썬 코드를 Javascript로 변환해야 하는데, 아래 내용이 적용되어야 해.

그 다음 아래 내용에 유의해서 결과를 만들어줘.

1. 문제 해결 개요를 오늘 수업 주제 알고리즘 중점으로 정리해서 내가 복사 할 수 있는 Markdown 소스코드로 생성할 것
2. 문제 해결 개요는 `### 📌 문제 해결 개요` 형태로 제목을 제시해야 하고 간략하게 몇 문장으로 요약할 것
3. 문제 해결 개요의 하위 컨텐츠로 핵심 개념을 제시할 것
4. 핵심 개념은  `#### 🔹 핵심 개념`으로 제목을 표시하고 블럿 리스트 행태로 전달해야 함.
5. 문제 해결 개요의 하위 컨텐츠로 문제 해결을 위한 아이디어와 해결 방법을 넘버 리스트 형태로 제시할 것
6. 문제 해결을 위한 아이디어와 해결 방법은 `#### 🚀 해결 방법` 형태로 제목을 제시할 것
7. solution 함수의 구현 내용을 설명해야 함. solution 메서드를 정의해야 한다는 것은 이미 문제에 전제가 되어 있으므로 언급할 필요 없음.
8. 위 항목을 하나의 Markdown 소스코드로 제시해야 하며 제시되는 내용중에 굵게 표시되는 텍스트는 없어야 함.
9. Markdown 소스코드를 생성한 후에는 내가 업로드한 소스코드를 변경하지 말고 각 라인마다 주석으로 설명을 추가할 것.
10. 내가 올린 소스코드에 이미 주석이 작성되어 있는 경우 이미 작성된 주석을 무시하고 더 자세한 내용을 적용해야 함.
11. 반드시 내가 업로드 한 소스코드만을 활용해야 함.
12. 내가 업로드 한 문제 내용을 인용하지 말것.
13. 내가 업로드 한 코드의 개선사항을 제시할 필요는 없음
14. Markdown 설명 내용과 주석이 추가된 소스코드는 분리된 블록으로 제시할 것
15. 반드시 문제 해결 개요의 Markdown 소스코드와 주석이 추가된 소스코드를 내가 각각 복사할 수 있어야 함
16. Java의 경우 StringBuilder는 사용하지 말 것.
17. 주석은 소스코드의 오른쪽에 추가하지 말고, 소스코드의 윗줄에 표시할 것.
18. 내가 사용한 변수 이름은 그대로 유지하되, 카멜 표기법을 적용해줘. 단, 한글자로 된 변수명은 원본의 대소문자를 그대로 유지해야 해
19. 제공되는 컨텐츠의 어투는 높임말이 아닌 간략한 어투를 사용할 것.
20. 소스코드의 들여쓰기는 공백문자 4개 단위로 할 것
